## 🛠️ How It Works

### Data Flow

```
vehicle telemetry data
        ↓
  data_loader.py (validate & enrich)
        ↓
 decision_engine.py (score & recommend)
        ↓
dashboard_production.py (visualize & optimize)
        ↓
   Business Decisions
```

### Risk Scoring Algorithm

The `decision_engine.py` calculates risk using:

```
Risk Score = 40% × battery_risk 
           + 35% × utilization_risk 
           + 15% × zone_risk 
           + 10% × maintenance_risk
```

**Result: 0-100 score mapped to risk levels**

- 0-25: LOW
- 26-50: MEDIUM
- 51-75: HIGH
- 76-100: CRITICAL

## ⚙️ Configuration & Customization

### Risk Scoring Weights

Edit `src/decision_engine.py` to adjust weights:

```python
RISK_MODEL_WEIGHTS = {
    'battery_weight': 0.40,      # 40% battery impact
    'utilization_weight': 0.35,  # 35% utilization impact
    'zone_pressure_weight': 0.15,  # 15% zone pressure
    'maintenance_weight': 0.10,  # 10% maintenance due
}
```

### Alert Thresholds

Edit `ALERT_THRESHOLDS` in `decision_engine.py`:

```python
ALERT_THRESHOLDS = {
    'battery_critical': 15,
    'battery_low': 30,
    'idle_hours_threshold': 24,
    'maintenance_overdue_days': 30,
}
```

### Data Source

Edit `dashboard_production.py` line ~80 to change data source:

```python
vehicles = loader.load_from_sample()        # Sample data
vehicles = loader.load_from_csv('path')     # CSV file
vehicles = loader.load_from_api('url')      # Live API
```

### Current Capacity

- **Vehicles**: Up to 1,000 vehicles (tested)

### Optimization Tips

1. **Reduce cache time** for more frequent updates:
   ```python
   @st.cache_data(ttl=60)  # 1-minute cache
   ```

2. **Increase cache time** for larger fleets:
   ```python
   @st.cache_data(ttl=600)  # 10-minute cache
   ```

3. **Filter at load time** for massive fleets
