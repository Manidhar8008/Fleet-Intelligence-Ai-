    <div id="hybrid" class="tunnel-content">
        <h2 style="color: var(--micro-purple);">Tunnel B: Hybrid AI (Prophet + LSTM)</h2>
        
        <div class="qa-block">
            <div class="question">Q: Why combine Prophet and LSTM? Why not just use one?</div>
            <div class="answer-key">
                <strong>The Technical Defense:</strong><br>
                "Prophet is excellent at <strong>Trend and Seasonality</strong> (Linear) but fails at complex non-linear patterns. LSTM is great at non-linearities but needs massive data to learn basic trends.<br>
                <strong>My Architecture:</strong><br>
                1. Use Prophet to predict the baseline.<br>
                2. Calculate the <em>Residuals</em> (Error = Actual - Prophet).<br>
                3. Train LSTM on the <em>Residuals</em> only.<br>
                4. Final Prediction = Prophet Output + LSTM Output."
            </div>
        </div>
        
        <div class="qa-block">
            <div class="question">Q: How do you handle overfitting in the LSTM part?</div>
            <div class="answer-key">
                1. <strong>Dropout:</strong> I used `Dropout(0.2)` layers.<br>
                2. <strong>Early Stopping:</strong> Monitor `val_loss` and stop training if it doesn't improve for 5 epochs.<br>
                3. <strong>Scaling:</strong> Crucial! I used `MinMaxScaler` (-1, 1) on the residuals because LSTMs are sensitive to unscaled data.
            </div>
        </div>
    </div>

<pre>
def process_huge_file(file):
    for chunk in pd.read_csv(file, chunksize=5000):
        yield process(chunk) # Keeps memory O(1)

            <div class="answer-key">
                <strong>The 3-Level Optimization Plan:</strong><br>
                1. <strong>Analyze:</strong> Run `EXPLAIN ANALYZE` to see if it's doing a "Sequential Scan" (Bad) or "Index Scan" (Good).<br>
                2. <strong>Indexing:</strong> Add a Composite Index on the columns used in `WHERE` and `JOIN`.<br>
                3. <strong>Partitioning:</strong> If the table is massive (TB scale), I'd partition it by Date so the engine skips old data files entirely.

<script>
    function openTunnel(tunnelId) {
        // Hide all tunnels first
        const tunnels = document.querySelectorAll('.tunnel-content');
        tunnels.forEach(t => t.style.display = 'none');

        // Show the specific tunnel
        const selected = document.getElementById(tunnelId);
        if(selected) {
            selected.style.display = 'block';
            // Smooth scroll to it
            selected.scrollIntoView({ behavior: 'smooth', block: 'start' });
        }
    }
